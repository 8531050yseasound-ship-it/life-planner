# Life Planner — ふたりの人生プランナー 説明書

行きたい場所・人生のお金・幸福度を、ふたりで共有しながら計画するアプリです。
スマホ・PC のブラウザで動き、ホーム画面に追加すればアプリのように使えます（PWA）。

## できること

| タブ | 内容 |
|---|---|
| 👥 ユーザー | 画面上部でユーザーを切り替え。**プラン・家計簿はユーザーごとに別々**に記録（行きたい場所はふたり共有）。設定でユーザーの追加・改名・削除ができ、同期するとお互いの全ユーザー分が見られる |
| 🗺️ 場所 | Instagram や YouTube で見つけた店・場所を、Googleマップ URL・元投稿 URL・期待幸福度(1–10)付きで登録。行ったら「行った！」で日付・実際の幸福度・感想を記録。**マップ表示**（リスト⇄マップ切替、緑=まだ・柿色=行った）と **Instagram保存済み投稿の一括取り込み** に対応 |
| 🧾 家計簿 | 月ごとの収入・支出を記録。**月表示（例：2026年6月）をタップすると年月ピッカーが開き、任意の年月へ直接移動可**（前/次の月ボタンでの1ヶ月ずつの移動も従来どおり。折半タブの月移動も同じ操作）。**楽天カード・楽天銀行・楽天証券・ゆうちょ銀行の明細CSVを取り込み**（カードは利用額＝支出／返金＝収入、銀行は出金＝支出／入金＝収入、**自動ジャンル分けは「コンビニ」（セブン/ローソン/ファミマ/セイコーマート等）と「光熱費」（電気/ガス/水道）のみ**（誤分類が多いその他は未分類にして分類モードで手動仕分け。既存データも初回に、コンビニ・光熱費以外を未分類へリセット）、文字コード自動判定、二重取り込み防止、**楽天銀行の「楽天カードサービス」引き落としは楽天カード明細との二重計上を防ぐため自動除外**）。支出の記録は**検索（店名・費目名・取り込み元）／費目フィルタ／支出・収入の切替／並び替え（新しい/古い/金額/ジャンル順）**で絞り込め、ジャンル順では費目ごとに小計付きでグルーピング表示。上部に件数と合計も表示。手入力も可。手入力時に**「12か月で分割して考える」**にチェックすると、年払いの保険・税金やボーナスなどを、その支払いをした年（1〜12月）に月割り（1/12ずつ）で家計簿（KPI・円グラフ・予算実績・一覧）に計上。各行のジャンルは選択で変更できるほか、**「🗂 分類モード」**は、上のプールに未分類の明細カード（費目チップで表示元を切替可）、下に全費目の移動先マスを常時表示。カードを**長押し→ドラッグ&ドロップ**で移動先マスへ入れて手早く仕分けできる（スマホのタッチ操作対応・主に未分類→各費目）。**手動で分けた「店名→費目」は学習され、同じ名前の未分類明細も即座にまとめて移動、次回以降の取り込みでも自動分類**される（店名は支店・番号違いを吸収。学習は費目選択・手入力でも記録。未分類へ戻すと学習解除）。分類モードや支出入力の「**＋ 費目を追加**」から、**自分で名前を付けた費目を追加**できる（分類・円グラフ・予算に反映）。KPIに**収入・支出・収支**、**支出の内訳を円グラフ**表示、ジャンル別に**予算と実績**を比較 |
| 👫 折半 | パートナーと**折半（割合変更可）で払ったもの**を記録する独立した家計簿。**2人の呼び名は自由に変更可**（この折半タブ専用）。**毎月の固定費**（家賃・光熱費・サブスク等）を折りたたみカードに登録すると毎月自動で折半の支払いに計上（一覧にも表示）。立て替えた人・負担割合から「いまどちらが誰にいくら支払えばよいか」を自動計算。「精算する」で貸し借りを0にリセット（履歴・取消あり）。**個人の家計簿やお金プランとは完全に別データ**で、ふたり共有 |
| 💰 プラン | 年齢・収入・生活費・ライフイベント（結婚式、車、旅行…費用と幸福度付き）から資産を90歳まで自動シミュレーション。「つみたてNISAを組み込む」をオンにすると現金と証券の2本立てになり、積立は現金から支出、現金が足りない年は証券を自動で切り崩して**「何歳から切り崩しが必要か」**をグラフの縦線と🔔メッセージで表示（不要ならその旨を表示）。現金も証券も尽きる年齢は⚠で警告。**「毎月の生活費プラン」**で生活費をジャンル別（家賃・食費・光熱費…）に予算立てでき、円グラフで内訳を確認。「生活費に反映」をオンにすると内訳の合計が毎月の生活費として使われる。グラフに旗が立ち、つみたてNISAシミュレーション（いまの評価額・投資済み元本を起点に、月額×今から続ける年数×固定利回り・月複利で将来の評価額を計算。横軸は年齢、評価額と元本の推移グラフ）付き。利回り既定値18.8%は楽天・全米株式インデックス・ファンド（楽天VTI）の設定来実績（2017/9〜2026/7年率・円ベース）で、自由に変更可。**「🎓 奨学金の返済」**で返済残高・年利（無利子=0）・毎月返済額から完済時期（年齢）・返済総額・利息・残高推移グラフを表示し、貯蓄シミュレーションにも反映 |
| ✨ ふりかえり | 行った場所のしあわせ集計・ランキング、予定の「しあわせコスパ」（幸福度÷費用100万円） |
| ⚙️ 設定 | ふたりで同期（Firebase）、JSONバックアップの書き出し/読み込み |

データはまず端末内（localStorage）に保存され、同期を設定すると自動でふたりの端末に反映されます。オフラインでも使えます。

**画面はスマホ・パソコン両対応（レスポンシブ）**：スマホでは下タブ、パソコン（幅1024px以上）では左サイドバー＋2カラムの広々レイアウトに自動で切り替わります。

## Instagramの保存済み投稿を取り込む

Instagramは保存済みコレクションを外部アプリへ自動連携するAPIを提供していません（非公式な自動取得は規約違反でアカウント凍結のリスクがあります）。代わりに2つの方法を用意しています。

**A. 一括取り込み（公式データエクスポート）**
1. Instagramアプリ → プロフィール → ☰ → アクティビティ（またはアカウントセンター）→ **個人データをダウンロード**
2. 「一部の情報」→ **保存済み** にチェック → 形式は **JSON** でリクエスト
3. 届いたZIPを、アプリの 🗺️場所タブ → 「📥 Instagram」ボタンから選択（ZIPのままでOK）
4. 保存した投稿のURLとアカウント名が「行きたい場所」として一括登録されます（重複は自動スキップ。写真・場所名はエクスポートに含まれないため、名前は後から編集）

**B. 1件ずつ共有（Android）**
ホーム画面に追加したあと、Instagramの投稿 → 共有 → **Life Planner** を選ぶと、URL入り追加フォームが開きます（iOSはPWAの共有ターゲット非対応のため、URLコピー→貼り付けで）。

## マップ表示と位置の設定

🗺️場所タブで「マップ」に切り替えると、OpenStreetMap上に 緑=まだ / 柿色=行った のピンが並びます（検索・絞り込みも連動）。位置は次の順で決まります：

1. GoogleマップURLに座標が含まれていれば自動抽出
2. 「📍 位置をまとめて取得」で名前から検索（OpenStreetMap Nominatim・1件/秒）
3. 見つからない場所は「🎯 地図をタップして設定」で手動配置

## 銀行・証券の明細から家計簿をつける

**⚠ 自動取得はできません（安全上の理由）**：楽天銀行・楽天証券・ゆうちょ銀行は、外部アプリがログインして残高や明細を自動取得できる公式の仕組み（一般個人向けAPI）を提供していません。ログイン情報を預かる自動取得は、規約違反かつセキュリティ上の重大なリスクがあるため実装していません。代わりに、各社サイトからご自身でダウンロードした**入出金明細CSV**を取り込みます。

1. 各社でCSVをダウンロード
   - **楽天銀行**：PCでログイン → 入出金明細 → 期間指定 → 「CSV形式でダウンロード」
   - **ゆうちょ銀行**：ゆうちょダイレクト → 入出金明細照会 → 「データダウンロード（CSV形式）」（ZIPのままでOK）
   - **楽天証券**：入出金・振替履歴などから CSV ダウンロード
2. アプリ上部で**記録したいユーザーを選択**
3. 🧾家計簿タブ → 「🏦 明細を取り込む」→ どこの明細か選び（自動判定も可）、ファイルを選択
4. 出金は支出、入金は収入として取り込まれ、内容から自動でジャンル分けされます（各行の🗑・ジャンル選択で修正可）。文字コード（Shift_JIS/UTF-8）は自動判定、同じ明細の二重取り込みは自動スキップ

より手軽に自動連携したい場合は、マネーフォワード ME・Zaim などの家計簿アプリ（金融機関と正規に連携する電子決済等代行業者）の利用をご検討ください。このアプリはそれらの代替ではなく、手元のCSVを取り込む方式です。

## ユーザーごとの記録と同期

- 画面上部の「ユーザー」で記録する人を切り替えます（この選択は端末ごと。相手の画面は変わりません）
- **プラン（貯蓄・NISA・生活費・ライフイベント）と家計簿は、ユーザーごとに別々**に保存されます
- **行きたい場所リストはふたり共有**です
- 同期をオンにすると、両方の端末でお互いの全ユーザー分のデータを見て編集できます（新しい方の変更を採用してマージ）

## ふたりで同期するための初期設定（1回だけ・約5分）

Firebase（Google の無料クラウド）を使います。**どちらか1人**が以下を行います。

1. https://console.firebase.google.com を開き、Google アカウントでログイン
2. 「プロジェクトを作成」→ 名前は自由（例 `life-planner`）→ Google アナリティクスは**オフ**でよい
3. 左メニュー「構築」→「Firestore Database」→「データベースを作成」
   - ロケーション: `asia-northeast1`（東京）
   - **本番モード**で開始
4. Firestore の「ルール」タブを開き、以下に書き換えて「公開」:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /rooms/{room} {
         allow read, write: if true;
       }
     }
   }
   ```
5. ⚙️（プロジェクトの設定）→「全般」→ 下の「マイアプリ」→ ウェブアプリ **</>** を追加（名前自由、Hosting は不要）
6. 表示される `firebaseConfig = { … }` の **{ } の中身ごと**コピー
7. アプリの ⚙️設定タブに貼り付け → 「あいことば」を**生成**（または自分で長めに決める）→「接続する」
8. パートナーには次の3つを共有: **アプリのURL**・**同じ構成（手順6のコピー）**・**同じあいことば**。相手も設定タブに入力して「接続する」を押せば完了

> ⚠️ このルールは「あいことばを知っている人なら読み書きできる」設定です。あいことばは推測されにくい長いものにし、本名・住所・口座情報などは書かないでください。

## スマホで使う（ホーム画面に追加）

アプリをインターネット上に置く必要があります（後述のGitHub Pages推奨）。開いたら:

- **iPhone/iPad**: Safari で開く → 共有ボタン → 「ホーム画面に追加」
- **Android**: Chrome で開く → メニュー ⋮ → 「ホーム画面に追加」（または表示されるインストールバナー）

以後はアイコンから全画面のアプリとして起動し、オフラインでも開けます。

## 公開（配布）の方法

- **GitHub Pages（推奨・無料）**: このフォルダ（`index.html`, `manifest.webmanifest`, `sw.js`, `icons/`）をリポジトリに置き、Settings → Pages で公開。URL をふたりで共有
- ローカルで試すだけなら: `node server.js` → http://localhost:8873

## バックアップ

⚙️設定 → 「書き出す（JSON）」で全データをファイル保存できます。「読み込む」では新しい方のデータを採用してマージするので、上書きで消える心配はありません。

## ファイル構成

```
life-planner/
├─ index.html          … アプリ本体（これ1つで動く）
├─ manifest.webmanifest … PWA設定（ホーム画面追加用）
├─ sw.js               … オフラインキャッシュ
├─ icons/              … アプリアイコン
└─ server.js           … ローカル確認用サーバー（配布不要）
```
